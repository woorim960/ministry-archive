# 구세군 마포영문 기획서 아카이브

구세군 마포영문에서 준비한 기독교 콘텐츠 기획서를 기록하고 공유하는 반응형 웹서비스입니다. 현재는 방탈출 기획을 중심으로 운영하며, 공개 기획서 읽기 화면과 운영자용 글쓰기 스튜디오를 포함합니다.

## 주요 기능

- 데스크톱·태블릿·모바일 반응형 기획서 아카이브
- 제목과 본문 통합 검색
- 저장·공개 상태가 구분되는 운영자 글쓰기 화면
- 실제 공개 화면과 동일한 실시간 미리보기
- 좌측 기획서 목록과 우측 미리보기 독립 접기
- Markdown 기반 제목·목록·인용·확장 블록·진행자 참고
- 드래그한 글자에 굵게·기울임·취소선·색상·배경색 적용
- 이미지 업로드와 본문 내 드래그 배치
- 유튜브 링크 임베드
- 대표 이미지 미등록 시 브랜드 기본 이미지 자동 적용

## 자주 사용하는 작성 문법

```md
## 큰 제목

| 인용문

> 펼쳐볼 제목
  확장 영역 안에 들어갈 내용은 스페이스 두 칸 뒤에 씁니다.
  여러 줄도 모두 두 칸씩 들여씁니다.

:::note[blue] 진행자 참고
현장에서 진행자만 특히 확인할 내용을 씁니다.
:::

@youtube[영상 설명](https://youtu.be/유튜브영상ID)

`붉은 강조 박스`
```

글쓰기 화면의 서식 버튼을 사용하면 위 문법은 자동으로 입력됩니다.

## 로컬에서 코드 열기

필요 환경:

- Node.js 22.13 이상
- npm

압축을 푼 폴더에서 다음 명령을 실행합니다.

```bash
npm ci
npm run dev
```

터미널에 표시되는 로컬 주소를 브라우저에서 열면 공개 화면을 확인할 수 있습니다. 글쓰기 화면의 로그인·저장·이미지 업로드는 배포 환경의 사용자 인증과 데이터 저장소를 사용하므로, 완전한 기능 확인은 운영 사이트에서 진행하는 것이 가장 정확합니다.

품질 검사는 다음 명령으로 실행합니다.

```bash
npm run lint
npm test
```

## 주요 폴더

```text
app/            페이지, 글쓰기 화면, API
components/     공통 UI와 문서 렌더러
data/           기본 예시 기획서
db/             데이터베이스 스키마
lib/            Markdown 파서와 공통 함수
public/         이미지, 로고, 폰트
tests/          자동화 검사
```

## 이 대화에서 계속 공동 작업하는 방법

### 1. GitHub 연결 방식 · 권장

이 프로젝트를 본인의 비공개 GitHub 저장소에 보관하고 ChatGPT의 GitHub 연결을 사용하면, 이 웹페이지에서 같은 프로젝트를 계속 수정하면서 모든 변경 이력을 남길 수 있습니다. 언제든 ZIP으로 내려받거나 이전 버전으로 되돌릴 수도 있습니다.

### 2. ZIP 교환 방식

GitHub를 연결하지 않아도 작업할 수 있습니다.

1. 이 대화에서 받은 최신 ZIP을 로컬에 보관합니다.
2. 다음 수정 요청 때 최신 ZIP 전체를 다시 첨부합니다.
3. ChatGPT가 해당 버전을 수정하고 새 전체 ZIP을 제공합니다.
4. 받은 ZIP으로 로컬 폴더를 갱신합니다.

중간에 로컬에서 직접 수정했다면 반드시 수정된 최신 ZIP을 다시 첨부해야 서로 다른 버전이 섞이지 않습니다.
