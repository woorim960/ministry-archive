# 글쓰기 UX 개선 기록

## 목표

처음 사용하는 운영자도 설명서를 따로 읽지 않고 기획서를 작성하고, 필요한 경우 본문에만 집중하며, 공개 결과를 예측할 수 있는 글쓰기 환경을 만든다.

## 단계별 개선

| 단계 | 사용자 문제 | 구현 내용 | 상태 |
| --- | --- | --- | --- |
| 1. 집중 편집 | 좌우 패널이 본문 공간을 줄임 | 목록·미리보기 개별 열기/닫기, 한 번에 접는 집중 모드, 마지막 화면 구성 기억 | 완료 |
| 2. 서식 발견성 | Markdown 문법과 효과 위치가 어려움 | 글자·문단·삽입 도구를 묶은 고정 도구막대, 모든 기능의 마우스·키보드 도움말 | 완료 |
| 3. 확장 블록 | `>` 아래 내용의 포함 규칙이 불명확함 | 접기·펼치기 버튼 자동 입력, 제목 다음 줄 공백 2칸 규칙을 예시로 안내 | 완료 |
| 4. 미디어 | 영상과 이미지 삽입 결과를 예측하기 어려움 | YouTube 링크 검증·현재 위치 삽입·동일 미리보기, 이미지 드롭 업로드와 위치 이동 | 완료 |
| 5. 대표 이미지 | 이미지를 생략하면 빈 카드가 보임 | 아카이브 전용 기본 대표 이미지 자동 적용 | 완료 |
| 6. 작성 진입 | 하단의 작은 `관리` 링크만 있어 찾기 어려움 | 상단 `기획서 작성` 버튼, 모바일 고정 작성 버튼, 푸터 명칭 개선 | 완료 |
| 7. 초안 전환 | 새 글에서 기존 글을 열면 작성 중인 글이 사라진 것처럼 보임 | 저장 전 글도 고유 작업 항목으로 유지하고 `작성 중` 목록에서 즉시 복원 | 완료 |
| 8. 패널 조작 | 접기 버튼이 패널과 떨어져 있어 관계를 파악하기 어려움 | 좌·우 패널 제목 영역에 접기 버튼을 배치하고, 접힌 가장자리에 열기 버튼 제공 | 완료 |
| 9. 도구막대 스크롤 | 효과 도구를 좌우로 넘기면 편집기 전체가 함께 움직임 | 효과 도구 전용 가로 스크롤 영역과 편집기 가로 넘침 차단 | 완료 |
| 10. 앱형 모바일 UX | 작은 화면에서 목록·저장·공개 기능을 오가기 번거로움 | 기획서 목록 드로어, 배경 닫기, 하단 저장·공개 고정 액션, 안전 영역 대응 | 완료 |

## 검증 기준

- 목록과 미리보기를 각각 접었다가 다시 열 수 있다.
- 집중 모드에서는 편집창만 보이고, 종료하면 이전 패널 구성이 복원된다.
- 접기·펼치기 도움말에서 `>` 제목과 공백 2칸의 관계를 바로 확인할 수 있다.
- 올바른 YouTube 링크는 공개 화면과 같은 미리보기에서 재생 영역으로 보인다.
- 대표 이미지를 선택하지 않아도 목록과 본문에 기본 이미지가 보인다.
- 데스크톱과 모바일에서 `기획서 작성` 진입점을 첫 화면에서 찾을 수 있다.
- 저장하지 않은 새 기획서를 여러 개 만든 뒤 기존 글을 열어도 `작성 중` 목록에서 각각 복원된다.
- 효과 도구막대를 좌우로 움직여도 편집기와 페이지 본체의 가로 위치는 바뀌지 않는다.
- 모바일에서 기획서 목록은 화면 위 드로어로 열리고 저장·공개 버튼은 엄지손가락 범위에 유지된다.
