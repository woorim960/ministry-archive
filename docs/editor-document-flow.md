# 편집기 기획서 상태와 사용자 흐름

## 목표

사용자가 여러 기획서를 빠르게 오가더라도 내용이 사라지거나 목록 순서가 갑자기 바뀌었다고 느끼지 않게 한다. 목록은 탐색을 위한 안정적인 기준점으로 유지하고, 상태 변화는 순서 이동 대신 짧은 상태 문구와 표시점으로 전달한다.

## 목록 구조

| 영역 | 포함 항목 | 순서 규칙 | 표시 상태 |
| --- | --- | --- | --- |
| 작성 중 | 서버에 한 번도 저장하지 않은 새 기획서 | 최근 생성 항목을 위에 추가하고 이후 고정 | 저장 안 됨 |
| 전체 기획서 | 서버에 저장된 모든 기획서 | 최근 저장 순서이며 클릭·편집만으로는 바뀌지 않음 | 공개, 초안, 수정 중 |

저장된 기획서를 수정하는 동안에도 해당 항목은 `전체 기획서`에서 사라지지 않는다. 서버에 저장하지 않은 수정 내용이 있으면 같은 위치에서 `수정 중`으로 표시하고, 클릭하면 서버 원본 대신 작업 중 내용을 연다.

## 행동별 흐름

| 사용자 행동 | 편집 영역 | 목록 반응 | 데이터 처리 |
| --- | --- | --- | --- |
| 새 기획서 선택 | 새 빈 문서를 열고 제목에 초점 | `작성 중` 최상단에 한 번만 추가 | 기기 내 작업 저장소에 즉시 등록 |
| 다른 새 글 선택 | 선택한 작업 내용을 복원 | 선택 강조만 이동, 순서 고정 | 떠나는 글의 최신 입력을 보관 |
| 저장된 글 선택 | 저장본 또는 작업 중 수정본을 표시 | 선택 강조만 이동, 순서 고정 | 수정본이 있으면 그것을 우선 사용 |
| 새 글 첫 저장 | 서버가 반환한 정식 문서를 표시 | `작성 중`에서 제거하고 `전체 기획서` 최상단에 추가 | 임시 식별자를 서버 식별자로 교체 |
| 기존 글 수정 | 현재 문서에 변경 상태 표시 | 같은 위치에서 `수정 중`으로 변경 | 원본과 분리된 작업 사본 유지 |
| 기존 글 저장 | 저장 완료 상태 표시 | 성공한 저장 결과를 최상단에 배치 | 작업 사본 제거, 서버 목록의 동일 항목 교체 |
| 저장 실패 | 입력 내용을 그대로 유지 | 현재 위치에서 수정 중 상태 유지 | 기기 내 작업 저장소에 보관하고 재시도 제공 |
| 새로고침 또는 재방문 | 마지막 선택 문서를 복구 | 모든 작업 중 항목과 순서를 복원 | 선택 식별자와 작업 목록을 함께 복구 |

## 대표 이미지 규칙

- 목록 카드에는 대표 이미지가 없을 때 브랜드 기본 이미지를 사용한다.
- 상세 기획서와 편집기 미리보기에서는 대표 이미지가 없으면 이미지 영역 전체를 생략한다.
- 대표 이미지 제거 동작은 `기본 이미지 사용`이 아닌 `이미지 제거`로 표현한다.

## 서식 도구막대 규칙

- 도구막대만 가로로 스크롤하며 편집기·페이지의 가로 위치는 고정한다.
- 트랙패드와 터치 스와이프를 지원한다.
- 데스크톱에서는 도구 사이의 빈 영역을 잡아 좌우로 끌 수 있다.
- 도구막대 위에서 세로 휠을 사용하면 가로 이동으로 변환한다.
- 버튼과 파일 선택 요소를 누를 때는 드래그 동작을 시작하지 않는다.

## 완료 검증 기준

- 기획서를 클릭하거나 수정해도 작성 중·전체 기획서의 기존 순서가 바뀌지 않는다.
- 수정 중인 저장 글을 포함해 서버의 모든 저장 글이 `전체 기획서`에 한 번씩 표시된다.
- 새 기획서 버튼은 좌측 패널에 한 번만 표시된다.
- 대표 이미지가 없는 글은 목록에는 기본 이미지가 보이고 상세 페이지에는 이미지 영역이 없다.
- 우측 패널의 열림 여부와 관계없이 서식 도구막대의 모든 기능까지 가로 이동할 수 있다.
